1. O que é Hypertext Markup Language (HTML):
    Uma linguagem de marcação que foi criada especificamente para a exibição de aplicativos e páginas da web em navegadores da web. O HTML é como outras linguagens de marcação, anotando um documento e descrevendo sua estrutura, sintaxe e layout. Enquanto o HTML informa ao navegador o que exibir, ele não especifica os elementos de design visual, como fonte, cores e outros elementos de design. Esses são controlados por folhas de estilo em cascata ou CSS.

  2. Como e porquê surgiu o HTML?
    O HTML foi inventado por Tim Berners-Lee, um físico do centro de pesquisas CERN, na Suíça. Ele surgiu com a ideia de um sistema de hipertexto na internet. Hipertexto significa um texto que possui referências (links) para outros textos que podem ser acessados imediatamente.

  3. O que é eXtensible Hypertext Markup Language (XHTML)?
    É semelhante ao HTML 4.01 e é considerado como HTML definido como um aplicativo XML. Diferentemente do HTML, as páginas XHTML têm uma sintaxe estrita e precisam ser bem formadas para serem analisadas usando analisadores XML, diferentemente dos analisadores específicos de HTML mais branda.

  4. Qual a diferença de XHTML para HTML?
    A principal diferença entre XHTML e HTML, é que o primeiro é XML válido, enquanto o segundo possui uma sintaxe própria, mas ambos possuem sentido semântico. Para verificar se uma página foi construída corretamente no padrão XHTML, existem diversos validadores, como por exemplo: a W3C disponibiliza um validador online.

  5. O que é eXtensible Markup Language (XML)?
    A XML (eXtensible Markup Language, ou Linguagem de Marcação Estendida) é um subconjunto da SGML (Standard Generalized Markup Language, ou Linguagem de Marcação Padrão Generalizada) que permite que uma marcação específica seja criada para especificar idéias e compartilhá-las na rede. Ela tem as virtudes da SGML e da HTML sem qualquer das limitações óbvias.

  6. Qual a diferença de HTML para XML?
    A grande diferença entre HTML e XML é que o HTML descreve a aparência e a ações em uma página na rede enquanto o XML não descreve nem aparência e ações, mas sim o que cada trecho de dados é ou representa! Em outras palavras, o XML descreve o conteúdo do documento!
